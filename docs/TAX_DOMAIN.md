# 해외주식 양도소득세 도메인 규칙

## 과세 대상

- **해외주식 매도 차익** (국내주식은 대주주만 과세, 해외주식은 전원 과세)
- 과세 기간: 1월 1일 ~ 12월 31일 (역년 기준)
- 신고/납부: 익년 5월 (종합소득세 확정 신고)

## 세율

| 항목 | 금액 |
|------|------|
| 기본공제 | 250만원 (연간 합산) |
| 양도소득세 | 과세표준 × 20% |
| 지방소득세 | 양도소득세 × 10% (= 과세표준 × 2%) |
| **합산 실효세율** | **22%** |

```
과세표준 = max(0, 순양도차익 - 250만원)
예상세액 = 과세표준 × 22%
```

## FIFO 선입선출 원칙

한국 세법상 해외주식 양도차익 계산 시 **선입선출법(FIFO)**이 원칙이다.

### 계산 공식

```
양도차익 = 매도가(원화) - 취득가(원화) - 필요경비(수수료)

매도가(원화) = 매도 단가(외화) × 매도일 환율 × 수량
취득가(원화) = 매수 단가(외화) × 매수일 환율 × 수량  (FIFO 순서로 소진)
필요경비 = 매도 수수료(원화) + 매수 수수료(원화)
```

### 매도→매수 매핑 (FIFO Lot)

1매도 건 = N개 매수 건에서 소진. 예시:

```
매도: AAPL 15주 @ $200 (2026-03-15)
  → Lot 1: 매수 10주 @ $150 (2024-01-10)  ← 전량 소진
  → Lot 2: 매수  5주 @ $170 (2025-06-20)  ← 부분 소진 (잔여 5주)
```

## 잔고 기반 역산 알고리즘

### 문제

동기화 시 과세 연도만 가져오면 과거 매수 내역이 부족하다.

### 해법: 적응적 소급 (Adaptive Lookback)

```
필요 매수 수량(종목별) = 현재 보유 수량 + (과세연도 ~ 현재)의 모든 매도 수량
```

1. **현재 잔고 조회** (KIS API TTTS3012R) → 종목별 보유 수량
2. **과세연도~현재 매도 합산** → 종목별 매도 수량
3. **필요 매수 = 보유 + 매도** → 이만큼의 매수 내역이 DB에 있어야 함
4. **부족 시 과거 소급** → 1년씩 더 가져오기 (최대 2015년까지)

### 예시

```
AAPL: 현재 보유 10주 + 2026년 매도 5주 = 필요 매수 15주
      DB에 매수 10주만 있음 → 부족 5주 → 2024년 동기화 → 매수 5주 추가 → 충족
```

## 환율 규칙

| 출처 | 용도 | 비고 |
|------|------|------|
| **CTOS4001R `erlm_exrt`** | 정산 환율 (KIS 제공) | 가장 정확, 우선 사용 |
| **yfinance `USDKRW=X`** | 참고용 환율 | 15분 지연, cache.db 영구 캐시 |
| **서울외국환중개 매매기준율** | 세금 신고용 | 시스템 미연동 (면책 고지) |

- 체결일이 주말/공휴일이면 직전 영업일 환율 사용
- 과거 환율은 불변이므로 10년 TTL 캐시

## 수수료 규칙

| 데이터 소스 | 수수료 | 비고 |
|------------|--------|------|
| CTOS4001R | `frcr_fee1`(외화) + `dmst_wcrc_fee`(원화) | 실제 수수료 |
| TTTS3035R | 체결금액 × 0.25% (추정) | API에서 미제공 |
| 수동 입력 | 사용자 입력값 | 사용자 책임 |

## 시뮬레이션

현재 보유 종목을 가상으로 매도했을 때의 예상 양도세를 계산한다.

- DB에 저장하지 않음 (인메모리 계산)
- 실제 당해 매도 건 + 가상 매도 건 합산 → FIFO 계산
- 실제 분과 가상 분을 구분하여 결과 반환

## KIS API 체결내역 조회

| TR_ID | API | 특징 |
|-------|-----|------|
| CTOS4001R | 해외주식 일별거래내역 | 환율+수수료 내장, 기간별 조회 |
| TTTS3035R | 해외주식 주문체결내역 | 기간별 조회, 환율 미제공 |
| JTTT3001R | 해외주식 당일체결 | 당일만 (양도세에 부적합) |

3단계 fallback: CTOS4001R → TTTS3035R → 로컬 DB orders
